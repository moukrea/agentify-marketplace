## Findings

### F-101: moderate — LOOP_PROMPT.md uses v1 verdict vocab; CONVERGED gate unreachable

`plugins/agentify/LOOP_PROMPT.md` lines 249, 358, 365 reference
`"ship"`, `"ship-after-fixes"`, `"do-not-ship"` — the v1
audit-review-schema vocabulary that ADR 0001 superseded. Because the
CONVERGED exit gate (§C7) checks `last_verdict == "ship"` and
`finding-schema.json` v2 (`schema_version: 2`) only permits
`"healthy" | "degraded" | "broken"`, a v2-conformant review can never
trigger CONVERGED. The loop will only terminate via STALLED, BUDGET,
REGRESSION, or SUBAGENT_FAILURE — never DONE.

- **Acceptance criterion:**
  `! grep -nE '"ship"|"ship-after-fixes"|"do-not-ship"' plugins/agentify/LOOP_PROMPT.md`
  exits 1 (no matches), AND the CONVERGED/PARKED gates reference the
  v2 vocab `"healthy"`/`"degraded"`/`"broken"`.
- **References:**
  - https://github.com/moukrea/agentify-marketplace/blob/main/plugins/agentify/LOOP_PROMPT.md#L249
    (fetched_at 2026-05-13T21:25:00Z; local grep)

### F-102: moderate — REVIEW_PROMPT.md verdict prescription drifted from v2 schema

`plugins/agentify/REVIEW_PROMPT.md:80` instructs the REVIEW subagent
to emit "Ship / ship-after-fixes / do-not-ship" verdicts. The
subagent's JSON output is consumed by `LOOP_PROMPT.md` and validated
against `finding-schema.json` v2; v1 verdicts will fail validation
and trigger SUBAGENT_FAILURE, halting the loop on every iteration.

- **Acceptance criterion:**
  `! grep -nE 'ship-after-fixes|Ship / ship' plugins/agentify/REVIEW_PROMPT.md`
  exits 1, and the verdict-paragraph prescription names the v2 vocab.
- **References:**
  - https://github.com/moukrea/agentify-marketplace/blob/main/plugins/agentify/REVIEW_PROMPT.md#L80
    (fetched_at 2026-05-13T21:25:00Z)

### F-103: moderate — /mkt-practice-evolve distillation pipeline unimplemented

`/mkt-practice-evolve` (and `/mkt-self-improve` phase 8) prescribes
"distil new content into
`plugins/agentify/practices/distillations/<source-id>/<date>.md`" but
`plugins/agentify/lib/practice_track.sh` exposes only
`fetch | list_sources | gc` — no `distill` subcommand. The
distillations directory does not exist. The 6 sources fetched cleanly
in the previous audit therefore have no distilled recommendations,
and the per-recommendation `adoption_check_command` workflow has
nothing to read. The skill's promise of "surface unadopted
recommendations as `category: practice-drift` findings" is not
deliverable today.

- **Acceptance criterion:**
  `bash plugins/agentify/lib/practice_track.sh distill <source-id>`
  exits 0 for at least one source, AND
  `plugins/agentify/practices/distillations/<source-id>/<date>.md`
  exists, AND that file conforms to the recommendation schema in
  `plugins/agentify/conventions/pinned-practices.schema.json`.
- **References:**
