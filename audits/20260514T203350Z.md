## Findings

### F-001 — major / ci-broken: `changelog-pr.yml` fails 100 % of push runs (bot branch already exists, `git push` non-fast-forwards)

The most-recent three `changelog-pr` runs on `main` (HEAD `707fadd`,
`a87de32`, `4f7a0df`) all fail at the same step ("Open PR if CHANGELOG
diff is non-empty"). The failure tail (run 25882708504, log line at
20:05:33.4148) is `! [rejected]        bot/changelog-2026-05-14 -> bot/changelog-2026-05-14 (non-fast-forward)`. Root cause: the workflow
computes the branch name as `bot/changelog-$(date -u +%Y-%m-%d)`
deterministically, so two same-day pushes to `main` both try to
`git push -u origin "$branch"` against a branch that already carries
the first push's commit. `concurrency.cancel-in-progress: true`
(`changelog-pr.yml:14-16`) prevents two runs from being _in flight_
simultaneously but does nothing about the second run's push collision
once the first run has already pushed and exited. The result: CI
shows green for `ci.yml` but the `main` branch's run-list reads 23
failures of 30 (one cancelled).

Fix shape (preferred): on push collision, `git push --force-with-lease`
to the dated branch (safe because the bot owns it exclusively), OR
append `-attempt-$(uuidgen | head -c 8)` to disambiguate, OR fetch +
rebase before pushing.

- Acceptance criterion: 10 consecutive `changelog-pr.yml` runs on
  `main` conclude `success` (or `skipped` for "no changelog diff;
  nothing to PR"), measured via
  `bash plugins/agentify/lib/git_host.sh ci_status main 30`.

### F-004 — moderate / practice-drift: `plugins/agentify/.claude-plugin/plugin.json` declares no `dependencies[]` despite Claude Code v2.1.110+ supporting tag-resolved semver dependencies

`plugins/agentify/.claude-plugin/plugin.json` (32 lines) declares
`name`, `version: "6.0.0"`, `skills: "./skills"`, and metadata fields
— but no `dependencies` array. Claude Code v2.1.110 added first-class
plugin-dependency resolution: `dependencies` accepts either a bare
string or an object with `{name, version, marketplace}`, where
`version` is a Node semver range (`~2.1.0`, `^2.0`, etc.) resolved
against tags shaped `{plugin-name}--v{version}` on the upstream
repository. The companion CLI helper `claude plugin tag --push`
generates the tag deterministically from `plugin.json`. The same docs
introduce `allowCrossMarketplaceDependenciesOn` for the root
`marketplace.json`, the `dependency-unsatisfied` /
`dependency-version-unsatisfied` / `no-matching-tag` error codes, and
`claude plugin prune` (v2.1.121+) for orphaned-dependency cleanup.

The marketplace's `bin/bump-version.sh` syncs
`plugin.json:.version` + `marketplace.json` and creates an annotated
tag, but the tag is shaped `vX.Y.Z` (release-style), not
`agentify--vX.Y.Z` (plugin-dependency-resolution-style). Today the
plugin has no downstream dependents, so no consumer breaks; the gap
is forward-looking: the moment a fleet plugin declares `"agentify"`
in its `dependencies[]`, version resolution silently falls back to
"latest available" because no `agentify--v*` tag exists.

- Acceptance criterion: `bin/bump-version.sh` produces both the
  release-style `vX.Y.Z` annotated tag AND the plugin-resolution-style
  `agentify--vX.Y.Z` tag in a single `--push` (or documents the
  separation explicitly), AND `plugin.json` carries an empty-but-present
  `dependencies: []` array as schema future-proofing.

### F-006 — polish / practice-drift: `/goal` could compose with `/agt-loop`'s parent orchestrator as a per-turn completion gate

**Reframed from moderate to polish.** The original F-006 framed
`/goal` as structurally distinct from `/agt-loop`'s "interval
polling". That framing was wrong on two counts: `/agt-loop` does
not poll on an interval (it iterates synchronously turn-to-turn),
and `/goal` cannot replace `/agt-loop` (it returns yes/no via a
small fast model — no subagents, no structured audit JSON, no
convergence tracking). What remains is a small composition
opportunity, not a moderate practice-drift gap.

`/goal` is a session-scoped wrapper around a prompt-based Stop
hook: after each parent turn, the small fast model evaluates the
condition against the conversation transcript and returns yes/no.
`/agt-loop`'s parent orchestrator already tracks `last_verdict`
and trend fields in `${state_root}/loop-state.json`; the
orchestrator currently checks exit conditions inline (verdict ==
`ship`, regression streak, no-progress streak, max iterations).
`/goal "${state_root}/loop-state.json's last_verdict == 'ship'"`
would let the small fast model fire when the verdict transitions,
cleanly exiting the parent loop without the orchestrator needing
to evaluate exit conditions in each iteration.

This is an enhancement, not a gap. The orchestrator works
correctly today; `/goal` composition would just remove the
inline exit-condition check.

- Acceptance criterion: either an ADR records the deliberate
  decision NOT to compose `/goal` with `/agt-loop` (with
  motivation), OR `/agt-loop`'s SKILL.md notes `/goal` as a
  complementary primitive and `LOOP_PROMPT.md` §C7 documents the
  composition pattern.

### F-008 — info / meta: audit's first pass filed F-005 / F-006 without reading `LOOP_PROMPT.md`; retraction record + procedural fix

The first pass of this audit shipped two practice-drift findings
(F-005 moderate, F-006 moderate) that proposed retiring or
replacing `/agt-loop` based on naming similarity with bundled
`/loop` and structural reasoning about `/goal`. Both findings
were filed WITHOUT reading the actual contracts they criticized:

- `.claude/skills/agt-loop/SKILL.md` (86 lines) declares the
  skill drives `LOOP_PROMPT.md` and uses
  `subagent_type: general-purpose` for fresh-context REVISE and
  REVIEW per iteration. The skill explicitly notes "the mechanism
  differs because the use case differs: subagents give true fresh
  context per pass, which Stop-hook self-restart and `/loop`
  dynamic-mode wakeups do not".
- `plugins/agentify/LOOP_PROMPT.md` (527 lines, 27,580 bytes) is
  the in-session orchestrator: state machine at
  `${state_root}/loop-state.json` with fields `iteration`,
  `max_iterations`, `spot_check_counter`, `last_verdict`,
  `last_counts`, `prev_counts`, `no_progress_streak`,
  `caused_by_prior_revise_streak`, `agentify_md_sha`,
  `latest_review_path`, `latest_revision_path`, `parked_findings`;
  resume protocol reconciling state vs disk; AUTO-SEED for
  unseeded context bundles; dirty-tree gate; convergence
  detection across iterations.
- Bundled `/loop` is a polling scheduler (re-runs a prompt on a
  cron-shaped interval). `.claude/loop.md` body is capped at
  25,000 bytes — `LOOP_PROMPT.md` at 27,580 bytes is physically
  un-portable. Even if the body fit, the bundled skill has no
  subagent spawning, no state model beyond the cron schedule, no
  verdict tracking, no convergence detection.
- `/goal` is a per-turn condition evaluator that returns yes/no
  via the small fast model. It does not produce structured audit
  JSON, does not spawn subagents, does not track findings.

These three primitives compose; they do not replace each other.
F-005 is retracted entirely; F-006 is reframed from moderate to
polish (composition opportunity, not replacement). The headline
counts at the top of this audit reflect the post-revision state.

**Procedural fix.** `/mkt-self-improve`'s postflight gate currently
enforces evidence quality for cited URLs (FR-4: 20% sample
re-fetched with cited-keyword match). It does NOT enforce
evidence quality for cited in-tree file paths. Any practice-drift
finding that proposes retiring or replacing a plugin internal
should be required to cite the internal file at `file:line` and
quote at least one line of its contents — both verifiable
mechanically. This is the meta-lesson: bad audits don't merely
generate noise; they generate damaging signal when acted on.

- Acceptance criterion: `plugins/agentify/lib/mkt_self_improve_postflight.sh`
  gains an FR-8 gate: every finding with `category: practice-drift`
  whose `details` field references a `plugins/agentify/**` file
  path must include at least one corresponding `file:line` citation
  in `references[]` with a `quote` field of ≥20 characters drawn
  from that line. Failing the gate refuses the audit.
