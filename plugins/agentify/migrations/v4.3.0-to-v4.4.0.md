# Migration: agentify v4.3.0 → v4.4.0 (non-breaking)

This migration introduces the foundation pillars from the May 2026
architecture plan (governance, CI, secret-injection layer, git-host
abstraction, migrations infrastructure). All changes are additive — no
existing target file is renamed, removed, or restructured. Targets on
v4.3.0 can install v4.4.0 with zero behaviour change; the new abstractions
become available the moment a target opts into them via
`agentify.config.json`.

> **Audience.** Targets currently at v4.3.0. If you are bootstrapping a
> fresh repo on v4.4.0, skip this doc and follow the marketplace
> [`README.md`](../../../README.md) "Install".

---

## Breaking changes

No breaking changes in this release; all upgrades are zero-touch via
`/<p>-upgrade apply`.

## Manual steps

No manual steps required. The new abstractions are inert until configured.

## Auto-applicable steps

### Step A1: opt in to a non-default secrets provider

If your team uses `opaq`, set:

```sh
jq '.secrets = { provider: "opaq" }' agentify.config.json \
  > agentify.config.json.tmp && mv agentify.config.json.tmp agentify.config.json
```

The default remains `env` for full backward compatibility.

### Step A2: opt in to a non-default git host

If your repo lives on GitLab (or self-hosted Gitea/Codeberg/Forgejo), set:

```sh
jq '.git_host = { driver: "gitlab", endpoint: "https://gitlab.example.com/api/v4" }' \
  agentify.config.json > agentify.config.json.tmp \
  && mv agentify.config.json.tmp agentify.config.json
```

The driver auto-detects from `git remote get-url origin` when unset.

## Troubleshooting

- **Symptom:** `bash plugins/agentify/lib/secrets.sh check` fails with
  `opaq: opaq binary not found on PATH`.
  **Cause:** `secrets.provider: opaq` was selected but the `opaq` CLI
  is not installed.
  **Fix:** Either install opaq (https://github.com/moukrea/opaq) or
  revert to `secrets.provider: env`.

- **Symptom:** `git_host issue_list` returns `[]` unexpectedly.
  **Cause:** `gh auth status` reports no token in this environment.
  **Fix:** Run `gh auth login`, or export `GH_TOKEN` /
  `GITHUB_TOKEN`, or set `secrets.provider: opaq` and rerun via
  `bash plugins/agentify/lib/secrets.sh wrap git_host issue_list …`.
