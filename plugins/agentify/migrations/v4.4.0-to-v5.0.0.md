# Migration: agentify v4.4.0 â†’ v5.0.0 (BREAKING)

This release introduces **structural enforcement gates** that refuse to
let `/mkt-self-improve`, `/agt-prd`, `/agt-plan`, `/agt-tasks`, and any
`git push origin main` proceed without satisfying their documented
contracts. Motivated by the 2026-05-14 audit which demonstrated that
SKILL.md prose alone is not enforcement â€” the same model that ignored
"WebFetch every URL in context/*.md" also one-shot the entire lifecycle
and pushed 6 commits direct to main.

> **Audience.** Targets currently at v4.4.0. Fresh bootstraps on v5.0.0
> get the gates by default â€” skip this doc and follow the marketplace
> [`README.md`](../../../README.md) "Install".

## Breaking changes

| Change | Impact |
| --- | --- |
| `/mkt-self-improve` postflight refuses audits without `## Trend findings` section, sub-threshold `references[]` count, low-diversity hostnames, fetch-verification failures, or missing draft ADRs for new-domain citations. | Headless scripts producing audits must meet the substantive-research bar (PRD 0003 FR-2/3/4/5/7) or the audit file is deleted. No override flag, no opt-out env var. |
| `/agt-prd`, `/agt-plan`, `/agt-tasks` refuse to persist unless the model passes `--user-reviewed=<sha256>` to the matching preflight (or the transcript shows an `AskUserQuestion`/user message after the draft mtime). | Headless automation that called `task_backend prd_create` / `plan_create` / `task_create` directly must adopt the preflight invocation. See Step M2. |
| `git push origin main` (any refspec form) from the Bash tool is refused by a `PreToolUse` hook returning `permissionDecision: deny`. Bare `git push` from a working tree on `main` is also refused. | Maintainers and tenant operators must move to a feature-branch + PR workflow. Bot pushes to `bot/*` branches and feature pushes to non-main targets are unaffected. |

There is no compatibility shim env-var (e.g. `AGENTIFY_SKILL_GATES_PERMISSIVE`).
Major-version bump (4.4.0 â†’ 5.0.0) signals the break.

## Manual steps

### Step M1: Confirm your push workflow is feature-branch + PR

If your tenant repo's bot accounts or CI workflows currently `git push origin main` directly from the agent's Bash tool, switch to the canonical pattern:

```bash
git switch -c "moukrea/<scope>/$(date -u +%Y-%m-%d)-<slug>"
# ... make changes ...
git push -u origin "moukrea/<scope>/$(date -u +%Y-%m-%d)-<slug>"
gh pr create --base main --head "$(git rev-parse --abbrev-ref HEAD)" \
  --title "<scope>: <description>" --body-file <body>
# wait for CI green
gh pr merge --squash --delete-branch <pr>
```

Bot workflows (`changelog-pr.yml`, `audit-trend.yml`, `practice-evolve.yml`) already use `bot/<workflow>-<date>` branches and are not affected.

### Step M2: Adopt preflight in headless lifecycle callers

If your repo has scripts that invoke `task_backend prd_create` / `plan_create` / `task_create` headlessly (no human in the loop), wrap them with the preflight:

```bash
# OLD (refused under v5.0.0):
bash plugins/agentify/lib/task_backend.sh prd_create "$title" "$body"

# NEW:
body_sha=$(sha256sum "$body" | cut -d' ' -f1)
# Present $body to the human, collect approval (any mechanism), then:
bash plugins/agentify/lib/agt_prd_preflight.sh "$body" --user-reviewed="$body_sha" \
  || { echo "preflight refused; not persisting"; exit 1; }
bash plugins/agentify/lib/task_backend.sh prd_create "$title" "$body"
```

If you genuinely cannot have a human in the loop, the lifecycle artifacts will not persist. There is no opt-out. Reconsider whether headless lifecycle work is appropriate for your case.

### Step M3: Bootstrap pinned-practices.json for substantive Phase 8

If your `plugins/agentify/conventions/pinned-practices.json` has an empty `sources: {}` map, Phase 8 of `/mkt-self-improve` cannot produce adoption-drift signal. Seed it with at least the `authority_weight â‰Ą 4` sources from `sources.yaml` (the v4.4.0 â†’ v5.0.0 release of the marketplace did this in `4f7a0df`). The dynamic-minimum threshold (FR-3) draws from this map.

## Troubleshooting

- **Symptom:** `/agt-prd` reports `preflight: REFUSED â€” no interaction evidence`.
  **Cause:** The model called `task_backend prd_create` without first running the preflight + passing `--user-reviewed=<sha>`.
  **Fix:** Use the invocation pattern in Step M2. The sha must match the EXACT body file you are about to persist.

- **Symptom:** `git push origin main` from the agent returns `permissionDecision: deny`.
  **Cause:** Working as designed â€” direct push to main is refused.
  **Fix:** Branch first (`git switch -c moukrea/<scope>/<date>-<slug>`), push the branch, open a PR, merge via `gh pr merge --squash`.

- **Symptom:** `/mkt-self-improve` postflight reports `FR-3: references[] count=X < threshold=Y`.
  **Cause:** The audit cited too few references against the dynamic threshold (max of context-URL count, authority-source count).
  **Fix:** Do real research â€” WebFetch the URLs cited in `context/*.md` AND fetch from the `authority_weight â‰Ą 4` sources in `sources.yaml`. The threshold scales automatically with those lists.

- **Symptom:** `FR-7: <hostname> lacks a paired draft ADR`.
  **Cause:** The audit's Trend findings cite a URL whose hostname is not in `sources.yaml` or `context/*.md`, and no draft ADR exists.
  **Fix:** Generate the draft from `plugins/agentify/templates/lifecycle/add-source-adr.md.template`. Drop it at `decisions/drafts/draft-add-source-<host-slug>.md`. The maintainer reviews + moves it to `decisions/<NNNN>-...md` on accept.
