# Migration: agentify v5.0.0 → v6.0.0 (BREAKING)

This release brings the lifecycle skills into alignment with Claude Code's native plan-mode, refines the v5.0 discovery gate from per-citation friction to threshold-N accumulation, and adds an explicit Claude Code surface-evolution discovery step to `/mkt-self-improve` Phase 4. The release encodes the principle that the harness exists to *make Claude Code BETTER*, not to impede it.

> **Audience.** Targets currently at v5.0.0. Fresh bootstraps on v6.0.0 get the new defaults — skip this doc and follow the marketplace [`README.md`](../../../README.md) "Install".

---

## Breaking changes

| Change | Impact |
| --- | --- |
| `/agt-prd`, `/agt-plan`, `/agt-tasks` SKILL.md mandate `EnterPlanMode` at skill entry. The model drafts the artifact in Claude Code's native plan-mode for the read-only enforcement + canonical approval UI. After approval, the model writes the body and persists via the existing `task_backend <verb>_create` path. | Headless lifecycle callers that bypassed plan-mode in v5.0 keep working — `--user-reviewed=<sha>` flag is still accepted as a fallback path. But the documented happy-path is now plan-mode. |
| `mkt_self_improve_postflight.sh` FR-7 replaced: per-citation ADR draft requirement → accumulation-based. New-domain citations append to `plugins/agentify/practices/discovered-sources.jsonl` (committed, NOT gitignored); ADR drafts auto-generated only when a domain accumulates >=N citations across distinct audits. Threshold N defaults to 3, configurable via `agentify.config.json:.self_improve.discovery_threshold` (minimum 2). | Audits that cite novel sources are no longer penalized per-citation. Discovery is encouraged; durable interest signals (>=N audits citing same domain) drive ADR drafts auto-generated by the postflight from `plugins/agentify/templates/lifecycle/add-source-adr.md.template`. |
| `/mkt-self-improve` Phase 4 gains a sub-step: WebFetch `https://code.claude.com/docs/llms.txt` and diff against URLs cited in `plugins/agentify/context/claude-code-mechanics.md` + `verification-cookbook.md`. Newly-added Claude Code doc paths surface as Trend-findings entries. | The audit becomes the discovery loop for Claude Code evolution. The harness rides Claude Code's evolution rather than freezing against the v5.0 snapshot. |
| `session_interaction_check.sh` accepts `ExitPlanMode` tool calls (transcript scan, after draft mtime) as interaction-evidence. | Plan-mode-driven flows now satisfy FR-6 naturally. Existing `--user-reviewed=<sha>` and `AskUserQuestion` paths remain untouched. |

There is no compatibility shim env-var. Major-version bump (5.0.0 → 6.0.0) signals the break.

## Manual steps

### Step M1: Adopt plan-mode in any headless lifecycle callers

If your tenant repo has scripts that invoked `task_backend prd_create` / `plan_create` / `task_create` directly without plan-mode, two options: (1) keep using `--user-reviewed=<sha>` — the legacy v5.0 path still works as a fallback; (2) move to plan-mode — invoke `EnterPlanMode`, draft, `ExitPlanMode`, then persist. Required reading: `plugins/agentify/skills/agt-prd/SKILL.md` §"Plan-mode entry (mandatory)".

### Step M2: Review accumulated discoveries on first v6.0 audit

After the first `/mkt-self-improve` run under v6.0, inspect `plugins/agentify/practices/discovered-sources.jsonl` for accumulated citations. If your repo has prior audits that cited new-domain sources but couldn't produce ADR drafts under v5.0's friction, those citations are NOT retroactively imported.

### Step M3: Tune `discovery_threshold` (optional)

The default N=3 is a starting point. Tenants that want eager ADR-draft generation can set N=2; tenants that want quiet accumulation can set N=5+. Update `agentify.config.json` with `self_improve.discovery_threshold`.

## Troubleshooting

- **Symptom:** Audit lands but `discovered-sources.jsonl` has no new entries.
  **Cause:** The audit's `## Trend findings` section didn't cite URLs from out-of-curated-list domains. Only URLs IN the trend section count toward accumulation (URLs only in `references[]` do not).
  **Fix:** Make trend findings substantive — cite the specific upstream sources that informed each pattern.

- **Symptom:** Postflight reports `FR-7 (v6.0 accumulation): ... auto-generated`.
  **Cause:** A new-domain has crossed the threshold; the postflight generated `decisions/drafts/draft-add-source-<slug>.md` and is requesting maintainer review.
  **Fix:** Review the draft. Accept (move to `decisions/<NNNN>-...md` + update `sources.yaml`), defer (leave the draft in place), or reject (delete). Re-run the audit — once the draft exists OR is moved, the postflight passes.

- **Symptom:** `/agt-prd` preflight reports `REFUSED — no AskUserQuestion, ExitPlanMode, or user reply found`.
  **Cause:** The model called `task_backend prd_create` without first invoking `EnterPlanMode`/`ExitPlanMode`, AskUserQuestion, or passing `--user-reviewed=<sha>`.
  **Fix:** Use the v6.0 default path — invoke `EnterPlanMode` at skill entry; draft inside plan-mode; call `ExitPlanMode` with the body; the native approval UI runs; then persist.

- **Symptom:** Claude Code's `ExitPlanMode` hook misfires (#20397) and a plan is lost.
  **Cause:** Known upstream issue — when the user accepts the plan AND clears context, the `PostToolUse` hook does not fire. The harness's primary persistence is harness-owned (via `task_backend`), so the model still has the body — but the `capture-plan.sh` backup is silently skipped.
  **Fix:** The model should explicitly write the body to a temp file BEFORE clearing context. See `plugins/agentify/context/known-bugs.md` #issue-20397 for the full workaround.
