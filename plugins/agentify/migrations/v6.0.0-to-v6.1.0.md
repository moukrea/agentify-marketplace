# Migration: agentify v6.0.0 → v6.1.0 (non-breaking)

This release consolidates five iterations of the in-session AGENTIFY revise/review loop (iter-02 through iter-06 plus a manual closure pass) into a stable, shippable AGENTIFY.md prompt. Refinements are prompt-internal — the deployed harness (hooks, skills, lib scripts, settings) is unchanged. There are no breaking changes, no manual steps, and no schema changes. Installed v6.0.0 targets pick up the improvements at the next `/agt-upgrade apply` (or by re-running `/agentify`) with zero touch.

> **Audience.** Targets currently at v6.0.0. If you are starting fresh on v6.1.0, skip this doc and follow [`README.md`](../../../README.md) "Install".

---

## Breaking changes

No breaking changes in this release; all upgrades are zero-touch via `/agt-upgrade apply`.

---

## Manual steps

None. The release is prompt-only — `AGENTIFY.md`, `REVISE_AGENTIFY_PROMPT.md`, and `PATCH_LOG.md` changed; no hook, skill, or `lib/` script changed in behavior. Targets that pinned to v6.0.0 do not need to re-run any bootstrap step.

---

## Auto-applicable steps

### Step A1: Refresh the bundled prompt files via `/agt-upgrade apply`

`/agt-upgrade apply` re-renders the plugin's templated prompt files (`AGENTIFY.md` + the loop prompts) into the target with the new v6.1.0 text. Reversible (the prior copies live in git history). No state mutation; no settings change.

```sh
/agt-upgrade check          # confirms current ↔ available version
/agt-upgrade apply          # interactive walk through the prompt-file refresh
```

## Troubleshooting

- **Symptom:** `/agt-upgrade check` reports "current=v6.0.0, available=v6.1.0" but `/agt-upgrade apply` is a no-op.
  **Cause:** The plugin source mirror is stale; the local copy of `AGENTIFY.md` was already at the v6.1.0 content (e.g., this repo is the marketplace itself).
  **Fix:** No action — the in-place file is correct. Verify with the version-marker grep above.

- **Symptom:** Loop run after the upgrade hits REGRESSION on iter-01 with `caused_by_prior_revise: true` findings citing v6.0-era prose.
  **Cause:** A target-side fork of `AGENTIFY.md` that hasn't been re-rendered against the v6.1.0 plugin source.
  **Fix:** `/agt-upgrade apply` (or `git checkout plugins/agentify/AGENTIFY.md` if the target uses the source in place) to pick up the v6.1.0 prompt body.
