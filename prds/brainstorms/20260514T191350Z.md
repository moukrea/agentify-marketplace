# Brainstorm: v6.0 — plan-mode integration + discovery accumulation + Claude Code evolution surfacing

## Problem statement

Two refinements to v5.0.0 surfaced as user feedback after the release landed. (1) `/agt-plan` reinvents plan drafting in markdown despite Claude Code shipping a native plan mode (`EnterPlanMode` / `ExitPlanMode` + the `plansDirectory` config + the existing `capture-plan.sh` `PostToolUse` hook). The native flow has a polished approval UI, read-only enforcement during planning, and is the canonical Claude Code experience the harness should *ride*, not duplicate. (2) v5.0's FR-7 (audit cites a new domain in Trend findings → must produce a draft ADR for `sources.yaml`) created an anti-discovery incentive: every novel citation costs ADR friction, so the audit is pushed toward re-citing known sources. Both refinements are colocated in a single release because they share a meta-principle: *the harness is built to make Claude Code BETTER, not to impede it*.

Constraint surfaced during research: the `ExitPlanMode` PostToolUse hook is **unreliable** in 2026 — Claude Code has three open issues (#22343 wrong cwd; #20397 hook doesn't fire when user clears context post-approval; #21282 plan-mode transitions partially invisible to hooks). A design that depends on the hook firing for persistence will silently drop plans in the clear-context path. The plan-mode UI is solid; the hook layer underneath it is not yet production-grade.

## Open questions

- [ ] Plan-mode persistence: option 2 (UI-only + harness-owned persist) is the recommendation but the SKILL.md prose needs to be crisp enough that the model RELIABLY runs the persist step after approval. Should this be enforced by the FR-6 preflight (refuse persist unless EnterPlanMode/ExitPlanMode were called in the session — detectable via transcript parse)? Or trust the prose?
- [ ] Accumulation threshold N: 2, 3, or 5? Lower N = ADR drafts more eager; higher N = more lurking discoveries. Initial guess: N=3 (citation in 3 distinct audits indicates durable signal).
- [ ] When promoting an accumulated source to ADR draft, should the postflight ALSO add it to a "watchlist" sources.yaml entry with `authority_weight: 3` (sub-threshold) so it gets fetched even before the maintainer accepts the ADR? Closes the loop tighter.
- [ ] Plan-mode integration scope: only `/agt-plan`, or also `/agt-prd` and `/agt-tasks` in this release? Incremental adoption argues for `/agt-plan` only; user's framing was "Plan should use" (singular) suggesting plan first.
- [ ] FR-6 preflight: how does it detect plan-mode approval? Options: (a) transcript parse for `ExitPlanMode` tool call after the draft mtime; (b) a token file dropped by capture-plan.sh on hook fire (unreliable per #20397); (c) the model passes `--plan-mode-approved` explicitly after exiting plan mode.
- [ ] Claude Code llms.txt fetch: is there a stable index URL? (`https://code.claude.com/docs/llms.txt` mentioned in the docs index — need to verify cadence + content shape).

## Tentative direction

**Bundle alternative 2 (UI-only plan-mode + harness-owned persist) + D1 (accumulation file + threshold-N ADR drafts) + E1 (Phase 4 Claude Code index fetch) into a single v6.0 release.**

Three coordinated changes, one release narrative ("make Claude Code better"):

(a) `/agt-plan` SKILL.md mandates `EnterPlanMode` at skill entry. The model drafts the plan in plan-mode (read-only enforcement). After approval via the native UI, the model writes the plan body to a temp file, computes its sha, runs `agt_plan_preflight.sh --user-reviewed=<sha>` (which detects the ExitPlanMode transcript event AS the interaction proof), then calls `task_backend plan_create`. The existing `capture-plan.sh` PostToolUse hook stays in place as a best-effort backup capture under `~/.claude/plans/`. Reliability is harness-owned, not hook-owned.

(b) Soften FR-7 to **accumulation**: postflight extracts every new-domain citation from the audit and appends to `plugins/agentify/practices/discovered-sources.jsonl`. ADR draft is auto-generated only when a domain has accumulated ≥3 distinct-audit citations (N tunable via `agentify.config.json:.self_improve.discovery_threshold`). Audit narrative is FREE to cite novel sources; durable-interest signal drives ADR drafts; no per-citation friction.

(c) `/mkt-self-improve` Phase 4 gains an explicit "Claude Code surface drift" substep: WebFetch `https://code.claude.com/docs/llms.txt` (the index), compare against URLs cited in `plugins/agentify/context/claude-code-mechanics.md` + `verification-cookbook.md`, surface newly-added doc paths as Trend-findings entries. This is the meta-loop: the harness audit ITSELF should discover Claude Code evolution.

Versioning: v6.0 major bump. Breaking only in that `/agt-plan` now mandates plan-mode (headless callers must adopt). Discovery accumulation is non-breaking (FR-7 becomes more permissive, not stricter). Phase 4 substep is additive.
